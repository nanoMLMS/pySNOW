# Contributing

Thank you for your interest in contributing to `pySNOW`.

## Reporting Bugs

If you find a bug, please report it through the
[issue tracker](https://github.com/nanoMLMS/pySNOW/issues).

When possible, include:
- a clear description of the issue
- steps to reproduce it
- the expected and actual behavior
- relevant logs, error messages, or screenshots

## Suggesting Enhancements

Suggestions and feature requests are welcome.  
Please submit them through the
[issue tracker](https://github.com/nanoMLMS/pySNOW/issues).

## Submitting a Pull Request

To contribute code directly, please follow these steps:

1. Fork the repository.
2. Create a new feature branch (or work on the `develop` branch).
3. Write documentation for any new functions using the [NumPy docstring style](https://numpydoc.readthedocs.io/en/latest/format.html).
4. Ensure that your changes are properly tested. If possible, add tests compatible with `pytest` in the `test/` folder.
5. Submit a pull request targeting the `develop` branch, including a clear description of your changes.

The documentation is generated automatically using [Material for MkDocs](https://squidfunk.github.io/mkdocs-material/).
